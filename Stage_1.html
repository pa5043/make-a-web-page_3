Important Concepts

Lession 1: The Basics of the Web and HTML

How the Web Works

The world wide web (www) is a collection of hypertext markup language (HTML) documents. Hyperlinks, or links, make up the web creating a weblike environment consisting of text, images, videos and music along with other elements. A Web Page is a text document written in HTML. Web browsers can read these documents, interpret and display them based on the code instructions.

Major pieces of the web are: the user, user's computer plus their browser, the Internet, Hypertext Transfer Protocol (HTTP)and the servers that contain HTML document information.

HTML

Hypertext markup language (HTML) is made up of: text content, markup and references. The instructions are translated by the computer but are not seen by the end user, only the desired text, pictures and videos are visible.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, the coder tells the browser the type of each element by using HTML tags. This video explains the distinction well.

Why Computers Are Stupid

Computers cannot think, only respond to commands provided by the developer or coder. Anything missed in a line of code results in a error. This concept is also called writing with the correct syntax.

Inline vs Block Elements

HTML element are either inline or block. Block elements form an "invisible" box around the content inside of the box.

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the Browser)

Programmers cannot remember everything so there are multiple tools and techniques to assist with formatting and checking the code.

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on...

This displays in the browser as a series of nested boxes - multiple boxes within boxes.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

The different levels of indentation makes it easier to see element groupings. As a result, you must remember how many levels exist in order to close the element correctly.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition within the html code is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over increasing the number of errors due to manual entry. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.

Consistency: Using CSS when creating a Web Page improves the quality of the final product. You can establish the styles for each element and reuse the styles through out the Web Page. By avoiding repetion, it's now very easy to make changes to a style element.

CSS

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
